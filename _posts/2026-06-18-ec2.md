# Elastic Compute Cloud (EC2)
> It allows users to launch and manage virtual servers, known as instances, which can be configured with various combinations of CPU, memory, storage, and networking resources.

### Usage
> Apps, Web Servers and Dev Environments.

### Key Terms 
- Instances:- Basically a virtual machine running in AWS. you pick the OS, CPU, RAM. it can be Linux or Windows. Every instance has an ID like i-0abc123.
- Security Group:- It acts like a firewall for your instance. controls inbound (who can connect IN) and outbound (what the instance can reach OUT) traffic by port and IP.
> example: allow port 22 from 0.0.0.0/0 = SSH open to entire internet = bad
  
- Key Paris:- SSH login mechanism for EC2. AWS generates a .pem file — you keep the private key, AWS puts the public key on the instance. no password needed, just the key file. Don't allow logins via passwords.
- Elastic IP:- Simply this are just Private and Public Ip Address assign to EC2, that help us to connect with them.
- Instance MetaData Service (IMDSv):- a local HTTP endpoint only accessible from inside the instance at 169.254.169.254. it serves info about the instance — IAM creds, IP, userdata, etc.
> IMDSv1 = no auth needed → vulnerable to SSRF
> IMDSv2 = requires token first → harder to exploit

- IAM Instance Profile:- The way you attach an IAM role to an EC2 instance. when an instance has a profile → it gets temporary AWS creds automatically via IMDS. this is why stealing metadata creds is so powerful — the role might have S3, RDS, secrets manager access etc.
> Instance Profile → holds IAM Role → creds served at 169.254.169.254

# AWS Cli
```bash
# List EC2 Instances
aws ec2 describe-instances
aws ec2 describe-instances --region <region>
aws ec2 describe-instances --query "Reservations[*].Instances[*].[InstanceId,PublicIpAddress,State.Name,KeyName,IamInstanceProfile.Arn]"

# Detail Info
aws ec2 describe-instances --instance-ids [i-xxxxxxxxxxxxxxx]

# Describe Availabe zone
aws ec2 describe-regions
aws ec2 describe-availability-zones

# IAM Roles Attached
aws ec2 describe-instances --query "Reservations[*].Instances[*].IamInstanceProfile.Arn"
aws iam get-instance-profile --instance-profile-name [name]

# list Security Groups
aws ec2 describe-security-groups
aws ec2 describe-security-groups --query "SecurityGroups[*].IpPermissions[*].{From:FromPort,To:ToPort,CIDR:IpRanges}"

# IP Address of EC2
aws ec2 describe-instances --instance-ids <id> --query 'Reservations[*].Instances[*].PublicIpAddress'

# list AMI
aws ec2 describe-images --owners self

# EBS Volume 
aws ec2 describe-volumes
aws ec2 describe-snapshots --owner-ids self
aws ec2 describe-snapshots --restorable-by-user-ids all --owner-ids <Account ID>
aws ec2 describe-snapshots --owner-ids <account-id> --filters "Name=snapshot-id,Values=snap-*"
aws ec2 describe-snapshots --query "Snapshots[*].[OwnerId,SnapshotId,VolumeSize,Description]" --output text | grep  "Account ID"
dsnap list  &&  dsnap get <Snapshot ID>   # download snapshot

# Enumerate key paris
aws ec2 describe-key-pairs
```
